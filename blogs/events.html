Event-based Visual Odometry and Inertial Odometry

1. Event-based Visual Odometry (VO)

Event-based Visual Odometry (Event VO) is a paradigm that leverages asynchronous event cameras to estimate motion. [cite: 500, 534] Unlike traditional frame-based cameras, event cameras output brightness changes at pixel-level with microsecond resolution. [cite: 501, 535]

Key Features

  • High temporal resolution (microseconds) [cite: 502, 537]
  • Low latency and no motion blur [cite: 502, 538]
  • High dynamic range [cite: 502, 539]

Event Generation Model

Event cameras respond to changes in logarithmic brightness rather than absolute intensity. An event is triggered when the change in log intensity exceeds a predefined threshold. [cite: 503, 541, 542]

$$ \frac{d}{dt} \left( \log I(u,v,t) \right) \geq C $$

where \(I(u,v,t)\) is the image intensity at pixel \((u,v)\) and time \(t\), and \(C\) is the contrast threshold. [cite: 504, 544]

The polarity of the event is defined as: [cite: 505, 545]

$$ p = \begin{cases} +1 & \text{if } \frac{d}{dt} (\log I) \geq C \\ -1 & \text{if } \frac{d}{dt} (\log I) \leq -C \end{cases} $$

Thus, each event is represented as \(e = (u, v, t, p)\). [cite: 505, 547, 548] This representation makes event cameras invariant to absolute illumination and highly responsive to motion. [cite: 505, 549]

Sensor Used: Event VO commonly uses Dynamic Vision Sensors (DVS) or DAVIS cameras. [cite: 506, 551]

Limitations [cite: 507, 553]

  • Requires motion to generate events [cite: 507, 554]
  • Sparse data in static environments [cite: 507, 555]
  • Challenging outdoors due to noise and illumination variation [cite: 507, 556]
  • Works better indoors due to controlled lighting [cite: 507, 557]

2. Neuromorphic Sensor: DAVIS 240C

The DAVIS 240C sensor combines a standard frame-based camera with an event-based sensor. [cite: 508, 559]

Key Characteristics [cite: 508, 568]

  • Resolution: \(240 \times 180\) [cite: 508, 569]
  • Outputs both frames and events [cite: 508, 570]
  • High temporal resolution (events) [cite: 508, 571]
  • APS grayscale frames available [cite: 508, 572]

3. UZH DAVIS 240C Dataset

The DAVIS 240C dataset from UZH provides synchronized multi-modal data for visual-inertial odometry. [cite: 509, 574]

Available Data [cite: 509, 575]

  • RGB Images – Frame-based grayscale images [cite: 509, 576]
  • events.txt – Asynchronous event stream \((u,v,t,p)\) [cite: 509, 577]
  • groundtruth.txt – 6-DoF poses [cite: 509, 578]
  • camera_calib.txt – Intrinsic calibration parameters [cite: 509, 580]
  • imu.txt – IMU measurements (accelerometer + gyroscope) [cite: 509, 581]

The ground truth is represented as: \(\text{Pose} = (t_x, t_y, t_z, q_x, q_y, q_z, q_w)\). [cite: 509, 582, 583] This dataset enables both Event VO and Event-based Visual-Inertial Odometry (VIO). [cite: 510, 584]

6. Experiment 3: Event VO with Increased Temporal Window (100ms)

In this experiment, the temporal window for event accumulation is increased from 30ms to 100ms to improve feature correspondences. [cite: 513, 612]

Pipeline [cite: 514, 613]

  1. Read events from events.txt [cite: 514, 614]
  2. Accumulate events in two windows: Frame 1: \(t \in [0, 100]\) ms and Frame 2: \(t \in [100, 200]\) ms [cite: 514, 615, 616, 617]
  3. Generate time surfaces for both frames [cite: 514, 618]
  4. Detect features on Frame 1 (e.g., corners or high gradient regions) [cite: 514, 619]
  5. Track features to Frame 2 using SSD-based patch matching [cite: 514, 515, 620]
  6. Normalize matched keypoints using camera intrinsics: \(x_n = K^{-1} x\) [cite: 515, 621, 622]
  7. Estimate Essential Matrix using 8-point algorithm [cite: 515, 623]
  8. Recover pose \((R, t)\) from Essential matrix [cite: 515, 624]
  9. Compare with ground truth to compute RRE and RTE [cite: 515, 516, 626, 627, 628]

Key Insight [cite: 516, 629]

Increasing the temporal window results in: Higher event density, more feature correspondences, and improved epipolar consistency. [cite: 516, 630, 631, 632, 633]

7. Experiment 4: Robust Estimation using RANSAC and MAGSAC++

This experiment introduces robust estimation techniques to handle outliers in feature correspondences. [cite: 517, 644]

Pipeline [cite: 518, 645]

  1. Generate time surface images (same as Experiment 3) [cite: 518, 646]
  2. Detect and match features using SSD-based patch matching [cite: 518, 647]
  3. Normalize matched keypoints using camera intrinsics [cite: 518, 648]
  4. Apply robust estimation: RANSAC with \(k = 100\) iterations and MAGSAC++ with adaptive thresholding [cite: 518, 650, 651, 652]
  5. Estimate Essential Matrix for inlier correspondences [cite: 518, 653]
  6. Recover relative pose \((R, t)\) [cite: 518, 654]
  7. Compute RRE and RTE using ground truth [cite: 518, 519, 655]

9. Experiment 6: Direct Event-based Visual Odometry

Instead of relying on discrete features, this method directly uses intensity variations from event frames. [cite: 522, 682]

Pipeline [cite: 523, 683]

  1. Generate time surfaces for consecutive windows [cite: 523, 684]
  2. Initialize pose estimate \((R, t)\) [cite: 523, 685]
  3. Define photometric error between warped event frames [cite: 523, 686]
  4. Optimize pose by minimizing: \(\sum || I_1(u,v) - I_2(\text{warp}(u,v, R, t)) ||^2\) [cite: 523, 524, 687, 688]
  5. Iterate optimization using gradient-based methods [cite: 524, 689]
  6. Output optimized pose [cite: 524, 690]
  7. Compute RRE and RTE [cite: 524, 691]

10. Experiment 7: Direct Event-based Visual-Inertial Odometry

This experiment integrates IMU data with direct event VO for improved robustness. [cite: 525, 732]

Pipeline [cite: 526, 734]

  1. Read IMU data (angular velocity, acceleration) [cite: 526, 735]
  2. Integrate IMU to obtain motion prior [cite: 526, 736]
  3. Generate time surface images [cite: 526, 737]
  4. Use IMU prior to initialize pose [cite: 526, 738]
  5. Minimize photometric error with IMU regularization: \(E = E_{\text{photometric}} + \lambda E_{\text{IMU}}\) [cite: 526, 739, 747]
  6. Optimize pose \((R, t)\) [cite: 526, 740]
  7. Compute RRE and RTE [cite: 526, 741]

  • Improved stability in translation estimation [cite: 527, 758]

Conclusion

Event-based Visual Odometry demonstrates strong performance in high-speed and high dynamic range scenarios. [cite: 527, 528, 763, 764] While feature-based methods suffer from sparse correspondences, increasing temporal windows improves robustness. [cite: 528, 765] Direct methods and sensor fusion (VIO) further enhance accuracy. However, challenges remain in translation estimation due to limited epipolar constraints and noise sensitivity. [cite: 529, 766]
